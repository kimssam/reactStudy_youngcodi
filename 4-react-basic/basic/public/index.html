react에서는 기본적으로 화면 요소를 다양한 수준의 컴포넌트로 분할함으로써 재사용성과 유지보수성을 높인다.

html파일 안에서 javascript를 기술할 때에는 {}안에 기술한다. 함수명은 ()생략

스타일을 기술하는 방법으로 인라인, 내부선언방식, 외부선언방식을 모두 활용 가능하다.

1)인라인 방식은 중괄호를 {{}} 두 번 사용한다. 예)style={{color:"red"}}

2)내부선언방식은 객체형식으로 선언한다.:내부선언방식에서는 속성명에 -를 사용하지 않고 camelCase로 작성한다.

Props:properties 는 컴포넌트에 전달하는 매개변수와 같은 것으로 컴포넌트는 전달받은 Props에 따라서 표시하는 스타일과 내용을 변경한다.

State: 컴포넌트의 상태를 나타내는 값

1)useState : react hook에서 가장 많이 사용하는 기능으로 useState 함수를 사용해서 state를 다룬다. 리턴값은 배열 형태로 첫번째 인수로는 State변수, 두번째 인수로는 State를 업데이트 하기 위한 함수를 설정
ex) const [num, setNum] = useState();

2)useEffect : 리액트 훅 기능 중의 하나로 useEffect는 컴포넌트의 부작용을 제어하는 기능이다. 어떤 값이 변했을 때에 한해서만 어떤 처리를 실행하도록 해서 부작용을 막아준다.
ex) export const App=()=>{ useEffect(()=>{ alert(); },[num]); return( {//useEffect는 첫번째 인수로 화살표 함수, 두번째 인수로 반드시 배열로 지정} ) } =>이런 기능이 존재하는 이유는 컴포넌트가 재랜더링을 몇번이고 반복하다 보면 state가 많을 때 재랜더링 횟수도 기하급수적으로 늘어나서 매번 재랜더링을 하면 시간을 낭비하므로 값이 변했을 때에만 실행하고 싶을 때 useEffect를 사용한다.

export의 종류
1)normal export
export const SomeComponent = () =>{}; import {SomeComponent as Some} from "./SomeComponent" 2)default export const SomeComponent = ()=>{}; export default SomeComponent; import SomeComponent from "./SomeComponent" import Some from "./SomeComponent"

camelCase :schoolOfRock, snakeCase : school_of_rock, PascalCase : SchoolOfRock, kebab-case : school-of-rock

리액트의 네이밍 규칙

  1. 폴더명 : 주로 소문자, kebab-case, camelCase를 사용
  2. 컴포넌트명 : PascalCase
  3. js 함수, 변수, hook : camelCase
  4. asset파일 : 소문자

====포인트 정리====

  • 리액트(JSX)규칙 : 리턴 이후는 한 개의 태그로 감싼다., 이벤트나 스타일은 캐멀 케이스로 작성한다. return 이후에 자바스크립트 코드를 쓸 때에는 {} 를 사용한다.
  • 함수로 컴포넌트를 만들고 조합해서 화면을 만든다.
  • 컴포넌트에 전달하는 인수와 같은 값을 Props라 부른다.
  • 컴포넌트가 가지는 다양한 상태를 State라 부른다.
  • 두 개의 리액트 훅(useEffect, useState)을 기억하자
  • 두 export 의 차이를 파악하자
  • 재랜더링하면 컴포넌트 처음부터 코드가 다시 실행된다.
